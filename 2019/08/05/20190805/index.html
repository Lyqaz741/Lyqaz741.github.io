程序插桩技术

插桩 属于动态测试，向被测程序中添加一些语句（如打印），以检查程序的执行路径、变量的取值等。
在实际测试中，长在如下位置设立探测点：1.程序块的第一个可执行语句之前2.循环语句的出入口3.条件语句的各条分支入口4.输入/输出语句之后5.函数、过程、子程序调用语句之后
断言：在程序中插入用来判断变量特性的语句（如判断输出是否正确），以证实程序运行的特性

逻辑覆盖法（发现错误的能力由弱到强）

语句覆盖（SC）:让被测代码中的每条可执行语句都至少执行一次
判定覆盖（DC）:对每个条件判断的真假两路分支各至少执行一次
条件覆盖（CC）：让条件判断中的每个最小的条件至少取一次真和一次假
条件判定覆盖（CDC）：同时覆盖条件和分支
条件组合覆盖（MCC）：覆盖所有逻辑条件的组合
修订判定条件覆盖（MCDC）

基本路径测试法

基本路径测试法的步骤：
1.根据程序或流程图绘制流图；
2.计算其环复杂性，此数字即为基本路径的条数。
3.确定一组独立的程序执行路径(即基本路径)；
4.为每条基本路径设计一个测试用例。

独立路径和环复杂性

独立路径是任何贯穿程序的、至少引入一组新的处理语句序列或一个新条件的路径；流图中，独立路径至少包含一条在定义该路径之前未曾用到过的边。
环复杂性以图论为基础，有三种计算方法：
1.V(G) = 域的数量
2.V(G) = 边数 - 结点数 + 2
3.V(G) = 判定结点数 + 1
环复杂性指示了独立路径的条数。
